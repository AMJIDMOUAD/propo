01 The Challenge
Data Residency

EU compliance is non-negotiable

Enterprise clients require data to stay within EU borders. Any architecture has to enforce this at the infrastructure level — not as a policy promise.

Repeatability

One-off builds don't scale

Manually configuring a new VPC for every client is slow and error-prone. The first build has to produce a template, not just an environment.

Operational Risk

Knowledge can't live in one head

If only the original builder can replicate the environment, every new client becomes a bottleneck. The next deployment needs to be doable without me.

02 Recommended Architecture

A single-tenant VPC, fully network-isolated per client, with traffic locked to the minimum path it needs at every tier — and nowhere else.

Internet reaches only the ALB. App nodes reach the database, AWS services, and external APIs — never the reverse.

Internet → ALB only ALB → App nodes only App nodes → DB only App nodes → NAT / Endpoints only

06 Components, Purpose & Benefit
ComponentPurposeBenefit
Application Load Balancer Single public entry point; routes traffic to healthy app nodes No app node is ever directly exposed to the internet
ECS Fargate (.NET / Docker) Runs 2+ containerized backend nodes without managing servers No EC2 patching; scales out per client without re-architecting
RDS PostgreSQL Managed, private database tier for application data Automated backups, patching, and isolation from the public network
NAT Gateway Lets private app nodes reach external APIs (OpenAI, ElevenLabs, Twilio) Outbound-only — nothing on the internet can initiate a connection in
VPC Endpoints (S3, SES) Private connectivity to AWS services without touching the public internet Lower latency, lower NAT cost, and traffic that never leaves AWS's network
Security Groups Tier-to-tier firewall rules enforced at the network layer A compromised tier can't reach further than its one allowed hop
Terraform Modules Codifies the entire VPC as reusable, parameterized infrastructure Next client's environment is a config change, not a rebuild
07 Risk Assessment & Mitigation
Risk

Data residency violation if traffic exits the EU region or routes through global edge services.

Mitigation

All resources pinned to a single EU region; AWS service traffic routed through VPC Endpoints, never public internet.

Risk

Configuration drift between client deployments over time, breaking repeatability.

Mitigation

Terraform state per client, version-controlled modules, and no manual console changes after initial build.

Risk

Lateral movement if one tier is compromised — app node reaching things it shouldn't.

Mitigation

Security groups enforce single-hop access per tier; no tier can reach further than its one required destination.

Risk

Team can't replicate the build without the original engineer involved.

Mitigation

Plain-English runbook treated as a core deliverable, validated through a live handover walkthrough.
